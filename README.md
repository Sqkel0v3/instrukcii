# База инструкций — как это устроено

## Структура

```
index.html        ← открывать этот файл в браузере. Меню + движок-обёртка.
style.css         ← общий дизайн для всех инструкций. Трогать редко.
engine.js         ← общая логика (шаги, развилки, прогресс). Не трогать при добавлении новых инструкций.
data/
  01-scan.js        ← одна инструкция = один файл
  02-rollers.js
  03-ebs.js
  00-TEMPLATE.js.txt  ← рецепт/шаблон для новой инструкции (с комментариями)
```

Всё работает локально, без сервера — просто открыть `index.html` двойным
кликом. Всю папку целиком можно переслать, положить на диск/портал.

## Как добавить новую инструкцию (31-ю, 32-ю и т.д.)

1. Скопировать `data/00-TEMPLATE.js.txt`, убрать `.txt`, назвать например
   `data/12-название.js`.
2. Заполнить по образцу: заголовок, шаги (`type:"step"`), развилки
   (`type:"choice"`), финалы (`type:"end"`). Комментарии в шаблоне
   объясняют каждое поле.
3. Если в шаге есть фото — вставить как `data:image/jpeg;base64,...`
   (как перевести картинку в такую строку — см. ниже) либо просто
   положить файл картинки рядом и указать путь, например `img: "data/img/12-1.jpg"`.
   Если фото несколько — используйте `imgs: [...]` вместо `img`, они
   лягут в ряд (на телефоне — друг под другом).
4. Если в шаге есть критичное предупреждение (риск поломки оборудования,
   действие, которое нельзя отменить) — используйте поле `danger` вместо
   обычного `note`. Оно рисуется красным, а не жёлтым.
5. В `index.html` дописать одну строку в конце списка подключений:
   ```html
   <script src="data/12-название.js"></script>
   ```
6. Открыть `index.html`, проверить, что новая карточка появилась в меню.

Дизайн, шапка, прогресс-бар, развилки — всё это уже работает само,
не нужно ничего верстать заново. Интерфейс адаптивный — открывается
и на телефоне, и на компьютере без доработок.

## Развилки — суть

Развилка (`type:"choice"`) — это просто вопрос с вариантами ответа,
и у каждого варианта — свой `next`, то есть свой путь дальше. Ветки
могут снова сходиться в одну точку (например, обе ведут к одному и
тому же финалу) или заканчиваться по-разному.

## Как перевести картинку в base64 (если понадобится вручную)

Через Python:
```python
import base64
with open("photo.jpg","rb") as f:
    print(base64.b64encode(f.read()).decode())
```
Полученную строку вставить в поле `img` как:
`"data:image/jpeg;base64,ВСТАВЛЕННАЯ_СТРОКА"`

Либо проще — не кодировать, а положить файл картинки в `data/img/` и
указать в поле `img` просто относительный путь к файлу.

## Если инструкций станет очень много (50+)

Меню сейчас группирует по `category`. Если категорий станет больше —
можно добавить поиск по названию (в `engine.js`, в `buildMenu`) —
дайте знать, добавим.
